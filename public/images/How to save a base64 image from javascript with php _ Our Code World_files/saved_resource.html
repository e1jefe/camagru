        That's not a brilliant idea, here you are sending the image as a string, and there are some limitations to the maximum input size ($_POST), you might have problem with image bigger than 8MB after base64 encoding (so almost any 1920*1080 uncompressed png). Sending the image as a file seems a better option as it allows bigger images and is less server-intensive. (As $_POST values are handled in-memory, if a hundred people send a 2MB file, the server will handle 2*1.3(base64 bloating)*100=260MB in memory...)
